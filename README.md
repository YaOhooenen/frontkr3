Отчет по выполнению практических работ №13-17
Практика 13: Service Worker
Реализована регистрация Service Worker, который кэширует статические ресурсы (HTML, CSS, JS, иконки) при первом посещении. Добавлен перехват fetch-запросов с использованием стратегии Cache First для статических файлов, благодаря чему приложение загружается и работает даже при полном отсутствии интернета. Заметки сохраняются в localStorage и остаются доступными офлайн. Service Worker также обрабатывает push-уведомления и клики по ним.

Практика 14: Web App Manifest
Создан файл manifest.json с указанием имени приложения "Мои заметки", иконок разных размеров (16x16, 32x32, 192x192, 512x512) и режима отображения "standalone". В index.html добавлены мета-теги для мобильных устройств: theme-color для Android, apple-touch-icon и apple-mobile-web-app-status-bar-style для iOS. Это позволяет установить приложение на устройство. При установке приложение запускается в отдельном окне без элементов браузера.

Практика 15: HTTPS + App Shell
Настроен локальный HTTPS-сервер с помощью mkcert для корректной работы Service Worker и push-уведомлений. Сгенерированы сертификаты для localhost, запущен сервер на порту 3002. Реализована архитектура App Shell: каркас приложения (шапка, меню) загружается моментально из кэша, а контент подгружается динамически через fetch. Переключение между страницами "Главная" и "О приложении" происходит без перезагрузки страницы.

Практика 16: WebSocket + Push
Подключена библиотека Socket.IO для двусторонней связи в реальном времени. Сервер запущен на порту 3001. При добавлении заметки клиент отправляет событие newTask на сервер, который рассылает taskAdded всем подключённым клиентам. Уведомление моментально появляется у всех клиентов в виде всплывающего окна в правом верхнем углу страницы.

После подписки через кнопку "Включить уведомления" пользователь получает системные push-уведомления Windows даже при свернутом браузере. Добавлена кнопка "Отключить уведомления" для отписки от push-рассылки.

Практика 17: Детализация Push
Добавлена форма для создания заметок с напоминанием, включающая текстовое поле и input с типом datetime-local для выбора даты и времени. На сервере реализовано планирование push-уведомлений с использованием setTimeout. При создании напоминания сервер вычисляет задержку до указанного времени и устанавливает таймер. В момент срабатывания таймера отправляется push-уведомление всем подписанным клиентам.

В уведомление добавлена кнопка "Отложить на 1 минуту". При нажатии на эту кнопку Service Worker отправляет POST-запрос на сервер. Сервер создает новую отложенную заметку с новым идентификатором и временем отправки через 1 минуту, а старое напоминание удаляется. Через минуту пользователь получает новое push-уведомление с текстом "Напоминание отложено". В самом приложении также появляется всплывающее уведомление об успешном откладывании. Напоминания работают даже при свернутом браузере.
